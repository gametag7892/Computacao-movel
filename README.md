# 📱 Estudos de Computação Móvel - Flutter

[cite_start]Este repositório contém os exercícios práticos desenvolvidos durante as aulas de **Computação Móvel** do curso de **Sistemas de Informação**[cite: 2, 977, 1468, 1835]. [cite_start]O objetivo principal é o domínio do framework Flutter e da linguagem Dart para o desenvolvimento de aplicações multiplataforma[cite: 1903].

---

## 🚀 Progresso por Roteiro de Aula

### 📝 Roteiro 1: Conceitos e Estrutura Inicial
* [cite_start]**Fundamentos Mobile**: Entendimento de interfaces touch, performance e responsividade[cite: 1848, 1855, 1891].
* [cite_start]**Flutter Engine**: Estudo sobre a engine **Skia**, que desenha a interface sem depender de componentes nativos[cite: 1962, 1964, 1971].
* [cite_start]**Configuração de Ambiente**: Instalação do Flutter SDK, Git e integração com VS Code e Android Studio[cite: 2005, 2015, 2051].
* [cite_start]**Widgets Básicos**: Diferenciação entre **StatelessWidget** (estático) e **StatefulWidget** (dinâmico)[cite: 2413, 2425].

### 🗺️ Roteiro 2: Navegação e Fluxo de Dados
* [cite_start]**Pilha de Telas**: Uso do **Navigator** para empilhar (`push`) e remover (`pop`) rotas da interface[cite: 1480, 1488, 1490].
* [cite_start]**Comunicação entre Telas**: Passagem de parâmetros via construtor e retorno de dados para a tela anterior[cite: 1584, 1630].
* [cite_start]**Assincronismo**: Implementação de `async`, `await` e manipulação de objetos `Future`[cite: 1733, 1741, 1750].

### 🏗️ Roteiro 3: Organização de Projeto e Listas
* [cite_start]**Arquitetura Relevante**: Separação de arquivos nas pastas `screens/` para telas completas e `widgets/` para componentes reutilizáveis[cite: 1007, 1010, 1021, 1022].
* [cite_start]**Componentização**: Criação de botões e textos personalizados para evitar repetição de código[cite: 1102, 1173].
* [cite_start]**Listas Dinâmicas**: Implementação de `ListView.builder` para renderização eficiente de grandes volumes de dados[cite: 1253, 1254, 1308].

### 🔄 Roteiro 4: Ciclo de Vida e Integração com API
* [cite_start]**Lifecycle de Widgets**: Domínio dos métodos `initState()` para inicializações e `dispose()` para liberação de recursos[cite: 98, 208, 257, 264].
* [cite_start]**Gerenciamento de Estado**: Uso correto do `setState()` para atualizar a interface após mudanças de dados[cite: 160, 196].
* [cite_start]**Consumo de API**: Integração com a **PokéAPI** utilizando o pacote `http` para buscar dados e `audioplayers` para reproduzir sons[cite: 398, 399, 491, 544].
* [cite_start]**Models e JSON**: Criação de classes de modelo e métodos `fromJson` para conversão de dados brutos da API em objetos Dart[cite: 691, 847, 869].
